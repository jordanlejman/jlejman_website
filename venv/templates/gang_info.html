About the Algorithm

How it works:

1. The player inputs any number between 1 and 500

2. The algorithm creates a set of 20 random guesses of numbers in that range. This is called a "Generation"

3. The algorithm checks how close each guess is to the player's number and gives them a "Fitness" score (higher numbers for closer guesses)

4. The algorithm takes the guesses with the 5 highest fitness scores and sets them aside, preparing to create a new "Generation"

5. The 5 best guesses are modified (4 times each) by adding a random number between -10 and 10 (here called "Mutation") creating a new, more refined generation of 20 guesses

6. This process is repeated until a generation containing the player's number is created


Okay, that's pretty neat, right? But what is it good for?

While guessing a number isn't that useful, the same basic steps can be applied to solve all kinds of problems. This game is just to show the concept, but I'll be adding cooler stuff with it later
